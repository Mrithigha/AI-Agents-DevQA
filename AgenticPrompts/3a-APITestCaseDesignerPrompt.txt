### PHASE 6 — Design API test cases

## Primary source — the story (always takes priority)
Design test cases based FIRST on the acceptance criteria in the Jira story.
For each AC write test cases covering:
- Happy path — valid inputs, expected successful response
- Negative paths — invalid inputs, missing required fields, wrong types
- Boundary conditions — empty values, max length, special characters
- Authentication — missing token, expired token, wrong role
- Error scenarios — not found, conflict, service unavailable

## Secondary source — PR diff (supplementary only)
After completing all story-based TCs, review the PR diff for:
- Any implementation
